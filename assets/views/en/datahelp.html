Panel

  1. Data update
    If you have a valid file proceed to point B. To work PATHFINDER you will need a file in JSON format which must be downloaded from the data server of data server of ExpoFinder. Identify yourself with access credentials or contact with the development team. Before saving it in the client’s disc take note of the PIN which PATHFINDER will supply. Without it you could not access to the download content.
    For reasons unrelated to the functional structure itself of PATHFINDER it is possible that sometimes the download occurs incorrectly or with some error, even that it will does not happen at all. In that case, try this operation again once a few minutes have passed and comunique the incidence to the development team.
  2. Database
    [IMPORTANTE: A REVISAR] Charge the JSON file locally saved and provide the correct PIN. After a brief work time (usually between 8 and 15 seconds) PATHFINDER will be ready to work.
  3. Exportación
    PATHFINDER allows you to download the data in XLS, TAB, CSV, XML or JSON format the raw data you use for your operation. Keep in mind that only the selected data will be exported through the active filter.
  4. Collections
    PATHFINDER deals with the concept of collection as a synonym of predefined filter. Development team generates these collections, which can be loaded directly from the server to limit the subset of data to be used for working. It is necessary to have an active Internet connection in order to use this feature.
  5. Reports
    For specific tasks in which the selected files are shown, PATHFINDER allows to extract the filtered data. Follow the steps shown below:
    1. Choose:
      1. The primary record type to be used as main in the report.
      2. The relational metadata field to be used as guide to find records related to main one.
    2. Choose the model [IMPORTANTE] that will be generated / to be generated:
      1. Analytical. Spreadsheet in CSV, format, compatible with Microsoft Excel™ or LibreOffice).
      2. Textual. Document in RTF format, compatible with Microsoft Word™ or LibreOffice.
  6. Estado de la base de datos
    [IMPORTANTE] If you have charge the specified file on point B, PATHFINDER will show the following information:
    If the file forementioned at step B has been loaded, PATHFINDER will show next info:
    1. Name of the file / file name
    2. Size of the file / file size (in bytes)
    3. ExpoFinder username which executed the file download / ExpoFinder identity used to download file
    4. Date of the file download / file download date
    5. IPv4 address from which the file download was made / IPv4 address used to download file
    6. Size of the database object in memory (in bytes) / database object size in memory (in bytes)
    7. Number of registers / records number
    8. Metadata number / metadata number
    9. Taxonomies number / taxonomies number
    10. Total number of elements (registers + metadata + taxonomies) / total number of elements (records + metadata + taxonomies)
    The More information… link will also be shown. Pressing it, PATHFINDER will open a pop-up window showing the recount of registers by type, its metadata and its taxonomy. In the latter cases, the absolute figure and the percentage with respect to the total are indicated.
    A More info… link will also be shown. By clicking on it, PATHFINDER will open a pop-up window showing a count of primary records by type, metadata and taxonomies included; last ones will show absolute figures and percentyage over total, too.

Listing

PATHFINDER shows a table with the filtered records. They appear sorted by type of record -> title of the record -> taxonomies associated with each record. By clicking on the title, the individual screen of the corresponding record will be shown.
PATHFINDER PATHFINDER muestra una tabla con los registros filtrados. Aparecen ordenados por tipo de registro → título del registro → taxonomías asociadas a cada registro. Al pulsar sobre el título, se mostrará la pantalla individual del correspondiente registro.

En la cabecera de la tabla figuran:

  1. número total de registros mostrados (según el filtro activo)
  2. página en curso y total de páginas
  3. número de registros por página (determinado en ajustes)
  4. enlace para establecer el filtro del sistema
  5. enlace para realizar una búsqueda simple
  6. enlace para limpiar el filtro y restablecer la base de datos a su condición original

En el pie de la tabla figuran:

  1. enlaces a páginas adyacentes a la que está en curso (determinado en ajustes)
  2. enlace para ir directamente a cualquier página
