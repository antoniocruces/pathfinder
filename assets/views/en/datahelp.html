Panel

It is the connectivity core of PATHFINDER. It makes possible update and load ExpoFinder data sets, export them in different formats using various criteria. Is possible, too, generate reports driven by links between records. Main significant elements are shown below.

  1. Update data

    If you have a valid file, go to next point. In order to work with PATHFINDER, you need a JSON formatted file, which must to be downloaded from the ExpoFinder data server. Identify yourself using your access credentials or contact development team to apply for them. Note you must write out and store in a safe place the supplied PIN before saving the file in the client computer disc. You can't be able to use it without this number.

    IMPORTANT. For reasons unrelated to the functional structure of PATHFINDER, it is sometimes possible that the download occurs incorrectly or carrying some error, or even it does not happen at all. If it is the case, retry the operation after a few minutes and report the incident to the development team.

  2. Database

    Load the locally stored JSON file and provide the appropriate PIN. After a short lapse (usually 8s -15s) PATHFINDER will be ready to run.

  3. Information

    If the file aforementioned in point B has been loaded, PATHFINDER will display the following info:

    1. file name
    2. file size (in bytes)
    3. name of the ExpoFinder user who downloaded the file
    4. file download date
    5. IPv4 address used to download the file
    6. database object size in memory (in bytes)
    7. number of primary records
    8. number of metadata
    9. number of taxonomies
    10. total items number (primary records + metadata + taxonomies)

    It will also show a count of primary records, their metadata and their taxonomies, showing, too, absolute frequence and percent.

  4. Export

    PATHFINDER allows download raw data set in TAB, CSV, XML formats. NOTE: Only filtered data will be exported.

  5. Collections

    PATHFINDER uses collection concept as a synonym of predefined filter. Development team generate these "collections" that can be downloadef from the application server and activated to set up a filter. Obviuosly is essential to be connected to Internet in order to be able to use this feature.

    When a collection is active, its title will be shown at Data menu pages header.

    IMPORTANT. Keep in mind that, because a collection is only a predefined filter, it is possible to modify it ir order to make it suitable for specific user purposes. If it is done so, the legend MODIFIED will be shown next to the title.

  6. Reports

    PATHFINDER allows to get filtered data gathered in a particular way following its relationship links. Follow next steps to do so:

    1. Choose:
      1. Primary record type used as main reference.
      2. Relational field used as link to find its relationships.
    2. Choose the model to be generated:
      1. Analitic. Sppreadsheet in CSV format, compatible with Microsoft Excel™ or LibreOffice).
      2. Textual. Document in RTF, format, compatible with Microsoft Word™ or LibreOffice.

Filter

IMPORTANT. Much of the computing power and discovery of new knowledge developed by PATHFINDER depends on the accuracy of the filtering process. Be careful when setting the conditions for not getting deceptive, inaccurate or confusing results.

BE MIND OF. Filter screen is a full-size pop-up window. It can be opened from any other PF screen. When closing it, screen from which filter was invoked will be maintained, and recalculated its results.

PF uses filer paradigm, typical of set theory and order theory, to narrow down the scope of the total set of records retrieved from ExpoFinder data server. The system will exclude from any calculation not matching records. The screen elements are as follows.

Inclusions & restrictions

They affect the filtered records subset when conditions have been applied.

  • Include. It adds to matching records those directly related to them matching checked types. For example, if conditions only affect exhibitions but actors are included, those related to each filtered exhibition will be added.
  • Restrictions. It limits included records to only those whose relational field is marked. As in the previous example, it is possible to select only actors acting as exhibition curators.

Filter panel

Each condition goes on a row, with features shown below.

  • Modifier. It selects a facet to use to validate conditions, as "year" for a date.
  • Field where search will be done.
  • Operator. The typical ones to compare textual or numeric values, having account, or not (as in contains and does not contains) uppercase and diacritics, plus is null and not is null, to find records having or not any value, and between, conceived to do comparisons for two sequential numeric values ("between 2010 and 2015" will match records in this range), although it also works with textual values.
  • Value. It is possible to get a matching value list clicking on icon.
  • Buttons for search (red background if not results found) and remove condition.

There is no limit in conditions number.

Operations

Buttons panel at filter window footer allowing:

  • To save or to load "collections", or frequently used filter definitions that user can store as a file at local disk.

  • Calculte. To do sequentially single calculation for each condition.

    IMPORTANT: PF needs to have done ALL single conditions calculation in order to set filter.

  • Filter. Evaluation of the conditions and generation of the filtered subset. When done a new button is visible, to show a graph to visualize links between primary records sat by conditions.
  • Clear. Initialization of the filter deleting the results subset.

BE MIND OF: As said, when a filter is set, PF shows a new button: Filter links, to open a pot-up window with a graph to visualize relations between primary records types.

Listing

PATHFINDER shows a table containing the filterd records, sorted by primary record type → record title → taxonomies. Clicking on title will show the primary record single screen.

IMPORTANT. Listing ONLY shows FILTERED RECORDS. Be mind of this if you can't find a particular one.

  1. Search & navigation

    Top area of the table allowing you to scroll through the data set using a slider or page by page basis. It also allows you to export filtered data in CSV, format, print table or search it for a row containing text or number.

    BE MIND OF. Table of results presentation model is the same in any operation done by PF. When result of an analytical operation is shown, over this place will appear, too, a minimal set of descriptive stats figures, including info about outliers, very important in discovering new knowledge process carried out by PF. It will appear too two links to show a couple of charts with sorted or normalized values (it means, as in a normal distribution).

  2. Table

    Filtered data sorted list. You can change order by clicking on each column header (ascending or descending one, A → Z, and vice versa).

  3. Rows

    By clicking on a primary record title, appropriate single record screen will be shown.

Stats

These operations make up the functional core of PF and allow to obtain new knowledge from the analysis of the qualitative data once quantified.

Groups

IMPORTANT. All calculations will be done ONLY over the FILTERED RECORDS SUBSET.

Calculates quantitative data through an aggregation function based on features or meta-information repeated in the different records, evaluating its frequency and distribution. Use this option to know the incidence of certain parameters in the complete sample of selected values. The highlights of the screen are as follows.

  1. Filter. It sets if all the filtered subset records or only meaningul ones will be shown (go to Documents to know more about outliers relevance in PF analytical processes). It is possible, too, to exclude those records having null or empty grouping values (not count ones).

  2. Selector to get query grouping fields. Writing part of a field name a matching list will be shown. Clicking on button, field will be added to groupers.

    IMPORTANT. Order is very important in groupers list, because it establish how the count will be done. For example, it is not the same thing group by actor's year of birth and actor's gender than vice versa. Each field has a symbol , and by clicking it field will be removed of the groupers.

Correlations

IMPORTANT. Probably this is, —with some particular operations with Charts, Networks and Maps— PATHFINDER's most powerful option for locating links and relationships between filtered subset data. Be mind of PF provides the tool, but the interpretation of results depends solely on the user. Is strongly recommended to read documentation in this regard.

Screen items are as follows.

  1. Filter. It sets if all the filtered subset records or only meaningul ones will be shown (go to Documents to know more about outliers relevance in PF analytical processes). It is possible, too, to exclude those records having null or empty grouping values (not count ones).

  2. Related fields according to the following scheme: 

    [Origin/Source] [Destination/Target]

    PF build a chain using relations between selected items, so that source be someway linked to target, and draws it, allowing edit it, too. For example, if source: actor → artwork authorship → artwork and target: exhibition → collector → actor is selected, PF will calculate all connections between both ends.

    IMPORTANT. Relation is calculated when both items, source and target, contain valid values. If it is not so, they will be marked as wrong. In addition, the calculated chain contains only filtered records.

  3. Calc result. Will be shown items below.

    1. Recalc and route graph buttons. The last one opens a pop-up window plotting how the different primary record tyles are related between in the calculated path.
    2. Textual description of route.
    3. Route. Is shown following the rules 
      [primary record 1] [relational field 1-n] [primary record n] [relational field n-n']...
      By clicking on any primary record label it will be deleted from the whole route and a recalc will be done.

    IMPORTANT. Deleting primary records makes calculated chain more relevant each time. In the previous example, chain be P artwork authorship A artwork included B catalog X collector P (see acronym equivalences by opening Legends). If the first one primary record type P is removed and B and X types too, it is possible to get how many times an artwork [A] is linked to an [P] actor acting a collector in different exhibitions.

Charts

They show visual representations of the quantified data from the filtered subset. In the paper "Graphs in Statistical Analysis", published in February 1973 by American Statistician, Frank Anscombe —later, in 1983, Edward Tufte so confirmed it in his book The Visual Display of Quantitative Information— demonstrated the importance of "visualizing" the data before, or in parallel, to the analysis process. PF tries to help in this regard by offering an overview of charts for the active and filtered records.

Screen items are as follows.

  1. Selection bar. Allows to choose the features of te chart to be shown.

    1. Table. Data to be charted. Can be of two kinds:

      • Query. Predefined selection of filtered data based on its characteristics (date, place, taxonomy…).

      • Statistics. Each of the analytical varieties used in Stats.

        IMPORTANT. A statistic will only be plotted if the corresponding modality has been previously calculated (Groups or Correlations).

    2. Filter. Each of the series or group of plotted data. In each of them data is grouped by a common feature. You can show all the series or choose them individually.

    3. Facete. Characterization of the data to be displayed.

  2. Result (chart). Chart types are as follows.

    1. Line / area. For queries. They can show:

      • Lines for the average of each serie.

      • Maximum value mark for each serie.

      • Series legend. By clicking on each serie label it will be or not visible.

      • Normalized sorting (normal distribution).

      • Linear regression. I shows dependency relationships between variables. When active, outliers are shown with a different symbol.

      • Exclusion or inclusion of null values.

Special chart layouts

If in selection bar, table section, option stats; correlations is choosen, PF offers 6 differents modes.

  1. Scatter. It uses the Cartesian coordinates to show the values of the pairs of variables in each series.

  2. Hierarchy. It shows a hierarchic tree (expressing dependence) in orthogonal format.

  3. Radial hierarchyl. It shows a hierarchic tree (expressing dependence) in circular format; the origin is the circle center.

  4. Treemap. Alternative to visualize the hierarchical structure showing also the quantities for each category through the size of its area.

  5. Sunburst. It presents the hierarchy through a series of rings, which are divided for each category node. Each ring corresponds to a level in the hierarchy, with the central circle representing the root node and the hierarchy moving outward from it.

  6. Wordcloud. "Cloud" formed by the titles of each primary record represented in the graph using a font size directly proportional to its total "weight".

Networks

Representation in a graph of the different relationships between distinct types of elements taken as a couple. Screen items are as follows.

  1. Selector bar. Allows you to choose both the presentation format and the data pair to present. Options are as follows.

    1. Type. Mode of presentation of the directed graph ( digraph). Any of the following values can be selected.

      1. Simple design. Unrestricted tree diagram, where each element appears linked by a line to those with which it relates.

      2. Force directed graph. Like the previous one, but the primary records are represented constrained by "gravitational forces", approximating or separating them, accommodating them too to the final size.

      3. Circular design. With the same principle of the two previous models, the primary records are arranged in a circle intersecting, if necessary, the lines that link them.

      4. Sankey design. Specific type of flowchart, in which the width of the joining lines is proportional to the amount of flow between two primary records.

      5. TAble design. Is plotted as a data table.

      6. Tree design. It is plotted as an unnumbered ordered list with indentation.

    2. Linked primary records, link type and relational fields. Selectables.
    3. Filter. Once the initial calculation is done, it is possible to recalculate the results by filtering only those matching the condition.
  2. Result. Shown in a similar way to the Charts ones.

Maps

PF uses the maps to show the accumulation, dispersion and distribution of data in a geographical context. The relevance of this type of presentation is well accredited in DIgital Humanities context, and it is in the spirit of PATHFINDER to normalize its use in the field of Digital Art History.

BE MIND OF: All PF calculations in Maps screen are done using only filtered data.

BE MIND OF: Obviously, only geolocatable data can be representen in a map.

Items in this screen are as follows.

  1. Base maps

    They are the "background" of the maps and provide cartographic information. PF allows you to select which one of them is active at any given time. Each one provides customized information highlighting or hiding some features. For example, those of "no labels" type (silent maps), which can make the global view clearer.

  2. Layers

    They are polygons or lines overlapping the active base map and delimiting some areas providing additional information. It is possible to activate more than one at a time, and the visible information obtained by clicking on them will correspond to the last one added.

    The list of available layers is below.

    1. Countries. Border line of the different countries of the world.

    2. European Union. Border profile of the different EU countries.

    3. Spain Spain. Regions ("comunidades autónomas"). Border profile of the different Spanish regions.

    4. Spain. Provinces (similar to counties). Border profile of the different Spanish provinces.

    5. Spain. Municipalities. Border profile of the different Spanish municipalities.

    6. Grid. Geographical meridians and parallels represented at gaps of 20 sexagesimal degreess.

    When a JSON data file two additional options will be shown.

    1. Groupes. All the filtered records are represented as clusters, represented by small colored circles (darker the more numerous the cluster). When you click on them or zoom in on the map, these circles are "broken down" and the individualized elements are displayed progressively.

    2. Heatmap. All the filtered records are represented in the form of continuous surface of graduated color by scale, showing greater or lesser concentration of elements in a specific area.

      0%100%

  3. Search

    An input text field allows to wrile all or part of a primary record title. PF will show a list of the matching ones.

    By clicking on one of them, it will ve drawn on map following the PF color code rules: for artworks, for catalogs, for companies, for entities, for exhibitions and for actors.

  4. Include

    When a search is performed, PF can show, too, additional info denoting its appropriate context. To do it, PF needs to know the buffer o "search scope range" as the radius of a circle from located point. Items shown at a dropdown list are shown below.

    1. Proximity in m. How much should measure the radius of the buffer used in the calculation. Note that the field accepts only positive integer numerical data, without decimal points, signs or thousands commas.

      PF will plot the located point and a semi-transparent gray circle bordered by dotted lines representing the scope covered by the buffer.

    2. Direct relationships. It shows lines joining the points representing the primary record located and those with a direct relation to it. This option is active regardless of whether a proximity radius in meters has been specified or not.

    3. Triangulated irregular network. Draw a mesh of three-sided polygons that link the localized point and those with a direct relation to it. It is usually abbreviated as TIN . The intensity of the inner semi-transparent color of each triangle indicates the relative importance of the connection between the various elements. This option is only active if a proximity radius in meters has been specified.

    4. Convex hull. It draws the a line connecting the more "outer" elements of the set of the found primary record and those with a direct relation to it. This option is only active if a proximity radius in meters has been specified.

    5. Envelope. It draws the rectangle connecting the more "outer" elements of the set of the found primary record and those with a direct relation to it at their NW-NE-SE-SW points. Do not confuse with envelope of a family of curves. This option is only active if a proximity radius in meters has been specified.

    6. Voronoi diagram. Also known as Thiessen polygons, it draws a triangular grid using Euclidean distance to calculate the interpolation of the located primary record those with a direct relation to it, based in the middle position between each pair of points. This option is only active if a proximity radius in meters has been specified.

    IMPORTANT. In Triangulated irregular network and Voronoi diagram, only the points being of the same primary record type tahn tthe found one will be linked.

  5. Queries

    Shown when at least a primary record has been found. They can be activated simultaneously or not, as is the case with Layers. It offers the following options.

    1. A selector for each query generated. The proximity search in meters includes three layers: located primary record, buffer included ones and the dotted circle itself.

    2. Clear. Remove all queries.

    3. Export. Generates a file in CSV format containing the primary records included in queries, whether they are showing or not.

  6. Time range and date selectors

    Those primary records having a temporal vector (e.g. actors or exhibitions) can be or cannot be shown at will. There are three controls to do so, by managing the "year" facet of those "temporalized" records.

    1. Temporal range. It uses a pop-up window to limit the older and the most recent date to be shown at map.

    2. Time lapse. By pressing the button triggers a system to automatically selects only one year in sequence from those included in the time range. That way, it is possible to watch the evolution of a phenomenon over time. It is particularly useful with an active heatmap layer. When the button is pressed again, the mechanism is sat off. It is possible to control the lapse in seconds for the year-by-year jump by means of Option menu, Settings submenu.

    3. Temporal selector. It acts, by means of a slider, filtering on map only the elements with the year shown by label.

    IMPORTANT. For all those records with a temporary vector having a start and end date (birth and death, opening and closing...) in Maps, only the start date is used to calculate the year facet.

Color codes

PATHFINDER uses a color coded system to identify different types of elements. Primary record types are identified as shown below.

Actor Person or entity interacting with catalogs or exhibitions acting as author, editor, illustrator, curator, loaner or other different roles.
Catalog Any article, book, web page or text in general used as guide or description for the visitor about a particular exhibition. Its "natural" utility is to store info about the artwork collection shown in exhibition. It may have a physical or virtual existence, being in this case the group of exhibited artwork.
Company Any organization whose relationship with entities, exhibitions, catalogs or actors could be defined STRICTLY in commercial or business terms. Its singularity, so, is not its public, private or mixed ownership, nor the fact of its aim includes or not profit motive, but the profit aim of the RELATIONSHIP.
Entity Any institution of a legal existence being used in ExpoFinder context both as RSS/Atom or HTML data source, whose results are consumed by the system to explore Internet to find temporary exhibitions, or as any other human group acting as organization or funding support for an exhibition or its catalog. May be, too, an institution linked to an actor acting as labor, research or academic reference.
Exhibition Any event showing publicly a set of artworks of an artistic nature or whose aim will be spread specific knowledge of the academic field of Art History.
Artwork Any manifestation of human activity by means of which the real is interpreted or the imagined is embodied using plastic, linguistic or sound resources. For the purposes of PF, it must be linked to actor/actors or catalog/catalogs.
