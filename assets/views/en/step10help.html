A: MAIN MENU.

Data
Groups the main operations of PF. It manages:
  • Download of data files in JSON format from ExpoFinder data server.
  • Load of working database from a JSON-formatted file downloaded from ExpoFinder data server.
  • Loaded and filtered data export.
  • Activation of predefined filters or collections, to work with a data subset.
  • Generation of reports based on filtered primary records, including, or not, metainfo and related ones.
Options
They allow to regulate the behavior of PF and adjust certain parameters of its operation.
Documents
Access to texts detailing the structure and functional principles of PF, the institutions involved in its development, the projects making use of its functionality and the operating manual.
Help
Pop-up window with relevant information for managing FP.

B: FILTER STATUS.

While no database is loaded ( JSON file) it shows the message No data. When loaded, filter is set up as the wholedata set (it means, there are no subsets). From then on, it will show the set and subset sizes as follows: [subset] / [set] ([percent]). It shows a pie microchart too, drawing the proportional sizes.

By clicking on this area it opens a pop-up window showing the status of the filter by type of primary record (or a warning message if a database has not been loaded).

C: SUBMENU.

When any main menu item is clicked (A zone) PF shows highlighted the item title and appropriate options, if necessary.

D: OPERATIONS.

The mos important zone. Any possible PF operation will have in the central area of the screen (this zone) the necessary elements to carry it out and configure it.

E: FOOTER.

Groups info about PATHFINDER status. Top to bottom they are as follows:

  • Upper line.
    • Application name, version and date. Clicking on it displays a pop-up window to expand those data.
    • Reference links (HD+, DOI, Github, iArtHis_LAB) and copyright.
  • Lower line.
    • Language change.
    • Page for contact ans legal info.
    • Chromatic PF legend.
    • Code libraries used in PF and load status of them.
    • Print and reload page.
    • Time spent in the most recent database operation, memory used by the app and maximum available to browser.

F: SEARCH.

When a database is loaded, PF allows make a search from here along all primary records dataset, looking for the whole or part of title. Case and diacritic insensitive. Shows a list with results. Clicking on any of them its single records screen will be shown.
