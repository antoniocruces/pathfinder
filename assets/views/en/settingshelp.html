Requirements

This page shows relevant information about the software and hardware of the client computer and shows some recommendations about minimum desirable requirements to optimize the user experience when running app.

BE MIND OF. Keep in mind that PATHFINDER is an application that try to maximize the power of the client computer, and barely lock server resources (data and application). It uses a browser to do so. Being Firefox and Chrome the recommended ones, nevertheless having one of them but outdated is not a good idea. The PF engine makes use of the latest technologies JavaScript and ECMAScript, and therefore you must ensure that you have browsers offering them.

TO THINK OF. It has been verified through extensive tests that PF can work correctly and with no drawbacks with only 4GB of RAM; even so, it is preferable to have 8GB as minimum, because currently it does not involve big money spending.

Settings

It allows you to configure the basic operating parameters of PATHFINDER. They can be modified by clicking on the desired option. Once changed, PF will store them in the client browser, and will be used in the following work sessions.

The screen has the following items.

  1. Error control.

    1. Error capture. PF will display a popup window when an error occurs.

    2. Detailed error messages. PF will include debug information in the error message. It will only be efective if Error capture option is activated.

    3. Show error in development console. PF will show error in browser console. It will only be efective if Error capture option is activated.

    4. Warning before page reload. PF will warn before executing a request to reload the page in the browser, in order to to prevent data loss. Recharge can thus be canceled.

  2. Limitations.

    1. Max load time for libraries (slow connections; in s). PF will warn if, trying to load a code library, exceeds this limit.

    2. Max items to show in descriptive stats. In analysis pages, section Statistics, PF shows a list of the most outstanding values ("top N values"). This parameter controls the number of those that will be displayed.

    3. Max of rows to show in a table (rows in a table, vertices and edges in a graph). Although the only PF limitation is determined by the amount of memory available on the client computer, it can improve the user experience if, in some particular circumstances, more restrictive limits to the recovered data are established. Strictly speaking they are not necessary.

  3. Tables & listings.

    1. Items per page. Page size (number of rows) for the tables shown on the screen.

    2. Z-score format. In the tables, where appropriate, PF shows the normalized value or z-score of each frequency. For more information about the standardized value or standardized unit, click here. This value can be displayed in numerical or graphic format.

  4. Color scale. PF adds to the tables that show frequencies a visual indicator of the relevance of each row with respect to the total. This indicator is shown by graduated colors.

Language

PATHFINDER is designed as a multi-language application as far as its interface is concerned (currently data is only in Spanish). This option (or the link at footer) allows to switch between the two available languages (at present): Spanish and English.

IMPORTANT. Keep in mind that, if you change language, PF will restart. If a database is loaded (JSON file), any operation carried out will be lost, although not the file itself. PF warns about database status when a language change is requested.
