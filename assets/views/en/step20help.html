1. DATA UPDATE.

Download of JSON file from data server. Each file is a "snapshot" of ExpoFinder database status at one point.

File is stored in the client local disk, encrypted using a one-time read-only PIN supplied by PF. It is strongly advised to keep it, because in case of forgetfulness or loss the file cannot be decoded.

This system allows successive "images" of ExpoFinder data to be stored and, therefore, make comparisons between.

2. LOAD FILE.

PATHFINDER will load in memory the JSON file stored in any local disk. PF will ask for the PIN to decrypt it before loading.

3. INFO.

Shown only if a JSON file is loaded in memory.

PF shows a pop-up window with specific data of the uploaded file, including the breakdown (in absolute figures and percentage) of each primary record, as well as taxonomies and metadata.

4. DATA EXPORT.

Shown only if a JSON file is loaded in memory.

Using filtered data, PF will build a file, downloadable in local disk, in one format ready to be read by other apps: TSV, CSV or XML. DAta will be exported in separate groups: primary records, taxonomies and metadata.

REMEMBER: only filtered data will be exported.

5. COLLECTIONS.

Shown only if a JSON file is loaded in memory.

It allows to use predefinited filters, stored at app server, to narrow out the main dataset through a group of conditions. If active, the name of the collection will be shown under data pages title.

6. INFORMES.

Shown only if a JSON file is loaded in memory.

Similar to export operation, but including, at will, relations for each primary record. Formats used are TSV and RTF. In the latter case, and given the complex nature of such files, the operation can take considerable time.

7. FILTER.

As stated in mathematical paradigm of filter, PF allows to select a subset of the database to deal with. Thereafter all operations made will only affect the aforementioned subset. Read docs to know more about.

IMPORTANT: if no filter is set, subset is equal to full database set.

8. LISTING.

PF shows all filtered records in a single paginated table exportable, sortable, printable and searchable by any text present in it. Presents a one primary record per row. It allows to access to any single record screen.

9. ANALYSIS.

The functional and operational core of PATHFINDER. It controls all the operations of generating new knowledge and discovering patterns from a triple approach analytical procedure:

  • Statistical: it applies methods of descriptive statistics and identification of anomalies from the quantification of qualitative data.
  • Graphic: draw diagrams, histograms and graphs based on quantified data.
  • Geographic: it shows dispersion or concentration in global maps and performs Boolean operations on geometric paths.
