Instructions

PATHFINDER uses filter, paradigm as in set theory. In short, it tries to select, including or excluding some particular features, a subset of the database records.

Note that, once a filter is applied to the data, it will remain active as long as there is no order against, and its presence affects ALL operations to be carried out by PATHFINDER.

Advanced filter

It allows to set successive and concurrent conditions in order to limit the scope of work. So:

  1. Set the restrictors for a condition. Note that only operator and value are rerquired.
    1. modifier: extract a particular element of some metainfos; for example, year or day of week of a date.
    2. metainfo: también denominado faceta en ciertos sistemas; determina sobre qué tipo de información se realizará la búsqueda; por ejemplo, lugar donde se celebra una exposición.
    3. operator: determines how a condition must be met; for example, that a record contains or does not equal a given value; keep in mind that such compliance, if no modifier or meta information is specified, will be verified on all records.
    4. value: to search (it can be a number, a date,…). BE CAREFUL: if you use between operator separate the values with the symbol "/" (bar). That is, if you want to search for a value between 2010 and 2011 write "2010/2011". Be conscious about a blue button () to the left of text input field to launch a particular search in order to get the more similar terms present in database matching the valueyou are trying to use. This kind of search is carried out on no taking account about letter case or accented chars basis.
  2. Click on the red button to the right to make PATHFINDER calculate the subsequent data subset. Note than button color will change to green when calculation is done.
  3. Add, if you want, new conditions to limit result as you need.
  4. At footer there is a purple-coloured button () designed to sequentially perform calculation for each single condition.
  5. When you think all the conditions are OK, press Set filter () blue button at footer. In a few seconds filter will be set.
  6. Above the first condition there are two dropdown selectors. Use them at will.
    1. Include. You can decide what primary record types must PATHFINDER include in the final result. By default only the records matching some of the set conditions will be included, but it is possible to force PATHFINDER to search for "concomitants" record types.
    2. Restrictions. Only the records matching the relational fields selected in this section will be included in final result.
  7. Next to it, PATHFINDER will show the figure of final result count and a tiny pie chart with the selected records percent. You can, too, get a table with result broken down by primary record types by clicking here.

NOTE: There is a list of valid metainfos, modifiers and operators available in in Terminology section.

Search

To make your search useful and effective, be sure to follow next advices.

  1. Write out all or part of text to search on.
  2. Select where you need the search will be carried out: a particular metadata kind or any metadata instead.
  3. Click the Search () blue button.
