Working…

PATHFINDER uses the paradigm of filter, like as in set theory. In short, it tries to select a subset of primary records by inclusion or exclusion of some features affecting to their titles, metadata or taxonomies.

IMPORTANT. When a filter has been set to data, it remains active until unset or until a new filter were set.

TO CONSIDER. The presence of a filter affects ALL operations carried on by PATHFINDER.

Filter definition

It allows to set conditions to limit the subset of primary records. The order in which the conditions are added does not affect the final result.

It is possible to add any conditions number by clicking on .

Only operator and value are mandatory in each condition. Any condition has the following items:

  1. modifier: extract one particular side of some metainformations; eg. year or weekday from a date.
  2. metainformation: also called facet; field to search for; eg. place where an exhibition is hold.
  3. operator: determines how a condition is to be met; eg. records containing or not being equal to some value; be mind of, if there is not modifier or metainfo specified, condition will be verified against the whole record set.
  4. value: text to search for (may be a number, a date,…).

    IMPORTANT. If operator between is used, values must to be separated with "/" (slash); eg. if you want to search values between 2010 and 2011 write "2010/2011".

    TO CONSIDER. Use of "contain" and "do not contain" operators implies that the search will be carried out not having mind of uppercase, lowercase or accented characters.

  5. match search: button shows a list of primary records, taxonomies or metainformations matching the text written in value text input element. Search will be performed using "contains" operator.
  6. partial result: is shown as a number of primary records matching condition, backgrounded as light red if 0 or light green if bigger.
  7. remove: delete condition.

Include and restrict

PATHFINDER performs filter calculation operation following a list of sequential steps:

  1. Calculates the primary records that SIMULTANEOUSLY meet ALL the conditions expressed (intersection between the subsets).
  2. Calculates ALL primary records related to the selected ones during step A matching the record types selected in Include menu.
  3. Final result will include unique records selected during steps A and B. But if one or more relational fields of the Restrictions menu is checked, only ones of selected set matching them will be included at end. For example, if primary actor records are included, and curator and collector are checked as relational fields, PATHFINDER will not select any actor unless included in step A conditions; if so, if all exhibitions in which Goya participates as artist are included but only curators and collectors are checked, PF will filter the exhibitions of the Aragonese painter, adding curators and collectors.

Operations

When all desired conditions are set, PATHFINDER performs the operations shown at the bottom of filtering window.

  1. Collections. A particular group of conditions, inclusions and restrictions is called collection. When ready (using the tools described above), the collection can be stored at local disk (client computer), and to be loaded when needed.
  2. Calculate. PF check all the conditions one each time and calculates its partial result.
  3. Filter and clear. Activation or removing filter for a particular collection.
  4. Filter links. When a filter is activated, this button will be shown. If clicked, it pops up a window plotting a directed graph of the primary record types and their relationships.
