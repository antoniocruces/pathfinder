ZONA A: MENÚ PRINCIPAL.

Datos
Agrupa las operaciones principales de PF. Gestiona:
  • La descarga de un archivo de datos en formato JSON desde el servidor de datos ExpoFinder.
  • La carga de la base de datos de trabajo usando un archivo local en formato JSON descargado desde el servidor de datos ExpoFinder.
  • La exportación de los datos cargados y filtrados.
  • El establecimiento de filtros predefinidos o colecciones, que generan un subconjunto de la base de datos cargada que cumplen determinadas condiciones.
  • La generación de informes para cada tipo de registro primario, de acuerdo con el filtro establecido e incluyendo —o no— otros registros relacionados, además de las correspondientes taxonomías y metainformaciones.
Opciones
Permite regular el comportamiento de PF y ajustar ciertos parámetros de su funcionamiento.
Documentos
Acceso a textos que detallan la estructura y los principios funcionales de PF, las institucioness involucradas en su desarrollo, los proyectos que hacen uso de su funcionalidad y el manual de operatoria.
Ayuda
Ventana emergente con información relevante para el manejo de PF:
  • Árbol de menús: representación jerárquica de la estructura de operaciones del sistema.
  • Preguntas frecuentes: trata de hacer frente a las dudas más habituales de los usuarios de PF mostrando sus respuestas.
  • Operatoria: diagrama de flujo que representa la estructura de la gestión habitual en el trabajo con PF.
  • ¿Cómo…?: apartado similar al de las preguntas frecuentes pero centrado en los objetivos finales del usuario ("qué debo hacer para...") y no en el trabajo con PF.

ZONA B: ESTADO DEL FILTRO.

Mientras no se haya cargado una base de datos (archivo JSON) se muestra el mensaje Sin datos. Tras la carga, el filtro se inicializa y queda establecido a la totalidad del conjunto principal de datos (es decir, no se establece ningún subconjunto). A partir de ese momento, mostrará los tamaños de conjunto y subconjunto de la siguiente forma: [subconjunto] / [conjunto] ([porcentaje]). También se muestra un micrográfico de tarta que expresa el tanto por ciento.

Al pulsar sobre esta zona se abre una ventana emergente que muestra el estado del filtro por tipo de registro primario (o un mensaje de advertencia si no se ha cargado una base de datos).

ZONA C: SUBMENÚ.

Cuando se pulsa cualquier elemento del menú principal (ZONA A) se muestran el título del elemento resaltado y, en caso necesario, las opciones correspondientes.

ZONA D: OPERACIONES.

Es la zona más importante del sistema. Cualquier operación que sea posible realizar con PF dispondrá en el área central de la pantalla (esta zona) los elementos necesarios para llevarla a cabo y configurarla.

ZONA E: PIE DE PÁGINA.

Agrupa información sobre el estado de PATHFINDER. De arriba abajo se muestran:

  • Línea superior.
    • Nombre de la aplicación, versión y fecha. Al pulsar sobre ella se despliega una ventana emergente que amplía los datos.
    • Enlaces de referencia (HD+, DOI, Github, iArtHis_LAB) y copyright.
  • Línea inferior.
    • Cambio de idioma.
    • Página de información legal y de contacto.
    • Colores empleados como referencia y símbolo en PF y sus equivalencias.
    • Bibliotecas (code libraries) usadas en PF y el estado de carga de cada una.
    • Recargar la página e imprimirla.
    • Tiempo empleado en la operación de base de datos más reciente y memoria usada por la aplicación con respecto a la máxima disponible a usar por el navegador.

ZONA F: BÚSQUEDA.

Cuando se encuentra cargada una base de datos a partir de un archivo local en formato JSON, PF permite buscar desde esta zona cualquier registro primario utilizando todo o parte de su título. La operación ignora el "case" (mayúsculas o minúsculas) y los diacríticos (tildes). Muestra una lista con los resultados obtenidos. Pulsando sobre cualquiera de ellos se despliega la correspondiente pantalla de registro individual.
