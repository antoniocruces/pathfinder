1. ACTUALIZAR DATOS.

Descarga de archivo JSON desde el servidor de datos. Cada archivo es una "foto fija" del estado de ExpoFinder en un momento dado.

El archivo se almacena codificado en el disco del ordenador cliente usando un PIN que es suministrado por PF y no puede ser cambiado. Se aconseja vivamente conservarlo, pues en caso de olvido o pérdida el archivo no podrá ser decodificado.

Este sistema permite guardar sucesivos "estados" de los datos de ExpoFinder y efectuar comparaciones.

2. CARGAR ARCHIVO.

PATHFINDER carga en memoria el archivo JSON conservado en disco que el usuario determine. PF solicitará el PIN para poder decodificarlo antes de cargarlo.

3. INFORMACIÓN.

Se muestra solamente cuando se haya cargado un archivo JSON en memoria.

PF muestra una ventana emergente con datos característicos del archivo cargado, incluyendo el desglose (en cifras absolutas y porcentaje) de cada registro primario, así como de las taxonomías y metadatos.

4. EXPORTAR DATOS.

Se muestra solamente cuando se haya cargado un archivo JSON en memoria.

PF genera un archivo que puede ser descargado en el disco local del ordenador cliente en formatos que pueden ser leídos por otras aplicaciones: TSV, CSV o XML. Los datos se exportan en grupos separados: registro primario, taxonomías y metadatos.

IMPORTANTE: solamente se exportarán los datos filtrados.

5. COLECCIONES.

Se muestra solamente cuando se haya cargado un archivo JSON en memoria.

Permite utilizar filtros predefinidos, conservados en el servidor de aplicación, que acotan el conjunto principal de datos de acuerdo a ciertos parámetros. Cuando una colección está activa, su nombre se muestra en la cabecera de las páginas de datos.

6. INFORMES.

Se muestra solamente cuando se haya cargado un archivo JSON en memoria.

Exporta el subconjunto de datos filtrados correspondiente a un tipo de registro primario (incluyendo o no un determinado tipo de registros relacionados) en formatos TSV o RTF. En este último caso, y dada la naturaleza compleja de tales archivos, la operación puede llevar un tiempo considerable.

7. FILTRO.

Siguiendo el paradigma matemático de filtro, PF permite seleccionar un subconjunto del conjunto principal de datos cargados. Todas las operaciones que se realicen a partir de entonces afectarán únicamente a dicho subconjunto. Consulte la documentación para conocer más sobre los filtros y la forma de construirlos.

IMPORTANTE: en caso de no haber ningún filtro establecido, el subconjunto es igual al conjunto completo de datos.

8. LISTADO.

PF muestra todos los registros filtrados en una única tabla paginada que puede ser exportada, impresa, ordenada y buscada por cualquier dato presente en ella. Mostrando a razón de un registro primario por fila, permite acceder a la ficha de registro individual de cada uno.

9. ANÁLISIS.

El núcleo funcional y operativo de PATHFINDER. Controla todas las operaciones de generación de nuevo conocimiento y descubrimiento de pautas y patrones a partir de un procedimiento analítico de triple enfoque:

  • Estadístico: aplica métodos de estadística descriptiva e identificación de anomalías a partir de la cuantificación de los datos cualitativos.
  • Gráfico: traza diagramas, histogramas y grafos partiendo de los datos cuantificados.
  • Geográfico: muestra la dispersión o concentración en mapas globales y realiza operaciones booleanas sobre trazados geométricos.
